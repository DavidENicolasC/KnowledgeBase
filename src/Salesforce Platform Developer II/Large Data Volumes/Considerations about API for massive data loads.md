- [[**SOAP API**]] - Optimized for real-time client applications that update a few records at a time. Requires developers and administrators to implement complex processes to upload data in bite-sized chunks, monitor results, and retry failed records. Acceptable for small data loads, but becomes unwieldy and time-consuming with large data sets.
- [[**Bulk API**]] - Based on REST Principles, designed to simplify and optimize the process of loading or deleting large data sets. Allows super-fast processing speed, reduced client-side programmatic language, easy-to-monitor job status, automatic retry of failed records, support for parallel processing, minimal round trips, minimal API calls, limited dropped connections and easy-to-tune batch size.
