* Replacing or adding another UI to your app.
* Providing a public-facing API to your logic - Assess parts of the code you would call to implement API.
* Scaling application logic via Batch Apex.
* Working with complex logic in your Visualforce controllers or @AuraEnabled methods - You can partition your code via Model-view-controller, a form of SOC.
* Making it easy for new developers to find their way around your code base.
