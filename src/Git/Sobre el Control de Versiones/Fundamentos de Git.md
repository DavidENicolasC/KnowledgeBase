La principal diferencia entre Git y cualquier otro VCS está en la forma de almacenar los cambios.

Los VCS tradicionales usan el [[Método para almacenar cambios de VCS tradicionales]].

Git usa el [[Método para almacenar cambios de Git]].

 La mayoría de las operaciones en Git sólo necesitan archivos y recursos locales para funcionar, sin la necesidad de estar permanentemente conectado a un servidor.

Git tiene tres estados principales en los que pueden estar los archivos de tu proyecto:
- [[Confirmado (committed)]]
- [[Modificado (modified)]]
- [[Preparado (staged)]]

Un proyecto de Git se divide en tres secciones:
- [[Directorio de Git (Git Directory)]]
- [[Directorio de Trabajo (Working Directory)]]
- [[Área de Preparación (Staging Area)]]
