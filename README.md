# Description

This is my personal Obsidian vault storing notes from concepts that I have been learning on all my academic life. Each one of the notes are constantly evolving, so concepts written here may change.
## Structure

The repo uses a combination of the following strategies:
- Linked notes - Relate particular concepts, avoiding to replicate concepts in other notes.
- Granular notes - Each note attends for only one concept. This concept can evolve, but the idea isn´t being hidden under other main idea in a larger note.
- Folder hierarchy - Each concept is related with others under a more general concept. This structure should not be taken to learn; is only useful for searching concepts and for start to learn/relearn an specific area of study.

## Use
As obsidian uses the `[...]` syntax to link notes, I recommend open this vault in the application rather than try start to learn simply by opening notes directly from the repo.
## About the structure

I've been looking for a personal method to efficiently learn avoiding knowledge gaps as possible.

The note-taking method always looks to me as a waste of time as you spend more time trying to write a condensed note for a concept than learning the concept. This also slowed to me to successfully study a complete material, as deadlines for home works and projects were always a priority. Students like me taken in account only a few concepts over all material that the book can provided to us, only to approve exam. Try to take notes reading all the book is still so slow, and read all the book without stop you to relate concepts can give to you without nothing correctly processed in your mind from you tried to learn.

More importantly, I'm trying find a way to relearn concepts using a structure that allows it after several months or years easily. What if I try to relearn only specific concepts about, let's say, Element attributes in AsciiDoc? I could write a dedicated note for all related to this concept, but I would already not need to learn all related to this concept. I could only want to learn where is being used; I can start in the "Main node", the defined entry point to learn about a concept, and go deeper on the concepts as much as I want, without get loosed by more information than already I don´t need to process.

This is because, after trying to take notes with Obsidian, I found useful to rapidly write and relate concepts while studying. This method forces to you to revisit concepts previously written, relating the new concepts to previously visited concepts for give to you a map of how concepts are related in an area of study.

This is not an effective method at all as I'm still trying to condense concepts before to write a note, and you require to refactor the notes constantly to avoid orphan and/or duplicated notes. It can´t cover all learning scenarios. But I'm feeling now how my notes have more value for me, instead of being deleted as their use were only to approve exams.
